# opnsense_trust_cert (Resource)

Manages an end-entity certificate in the OPNsense Trust store.

## Example Usage

```terraform
resource "opnsense_trust_ca" "example" {
  description = "Example CA"
  action      = "internal"
  key_type    = "2048"
  digest      = "sha256"
  lifetime    = "3650"
  country     = "US"
  common_name = "example-ca.internal"
}

# Server certificate signed by the CA
resource "opnsense_trust_cert" "web" {
  description  = "Web Server Certificate"
  action       = "internal"
  caref        = opnsense_trust_ca.example.ref_id
  key_type     = "2048"
  digest       = "sha256"
  cert_type    = "server_cert"
  lifetime     = "397"
  country      = "US"
  organization = "Acme Corp"
  common_name  = "www.example.com"
  altnames_dns = "www.example.com\napi.example.com"
}
```

## Schema

### Required

- `common_name` (String) Common name (CN) for the certificate.

### Optional

- `action` (String) Certificate action: `internal` (generate signed by CA), `external` (CSR only), `import` (import existing), `sign_csr`, `import_csr`, `reissue`, `manual`. Defaults to `internal`.
- `altnames_dns` (String) DNS Subject Alternative Names, newline or CRLF separated (e.g. `www.example.com\napi.example.com`).
- `altnames_email` (String) Email Subject Alternative Names, newline separated.
- `altnames_ip` (String) IP Subject Alternative Names, newline separated.
- `altnames_uri` (String) URI Subject Alternative Names, newline separated.
- `caref` (String) Reference ID (`ref_id`) of the signing CA. Required for `internal` and `sign_csr` actions.
- `cert_type` (String) Certificate type: `server_cert`, `usr_cert` (default), `combined_server_client`. Defaults to `usr_cert`.
- `city` (String)
- `country` (String) ISO 3166-1 alpha-2 country code (e.g. `US`, `CA`, `DE`).
- `crt` (String) Base64-encoded PEM certificate body. Required when `action` is `import`. Computed when `action` is `internal`.
- `csr` (String) Base64-encoded PEM Certificate Signing Request. Required when `action` is `sign_csr` or `import_csr`.
- `description` (String) Optional description for this certificate.
- `digest` (String) Digest algorithm: `sha224`, `sha256` (default), `sha384`, `sha512`. Defaults to `sha256`.
- `email` (String)
- `key_type` (String) Key type and size: `512`, `1024`, `2048` (default), `3072`, `4096`, `8192`, `prime256v1`, `secp384r1`, `secp521r1`. Defaults to `2048`.
- `lifetime` (String) Certificate validity period in days. Defaults to `397` (maximum accepted by modern browsers for server certificates).
- `ocsp_uri` (String) OCSP responder URI.
- `organization` (String)
- `organizational_unit` (String)
- `private_key_location` (String) Where to store the private key: `firewall` (default, store on the firewall) or `external` (CSR only, no key stored). Defaults to `firewall`.
- `prv` (String, Sensitive) Base64-encoded PEM private key. Required when `action` is `import`. Computed when `action` is `internal`.
- `rfc3280_purpose` (String) Extended Key Usage OID string (e.g. `id-kp-serverAuth`, `id-kp-clientAuth`). Computed by OPNsense based on `cert_type` when not explicitly set.
- `state` (String)

### Read-Only

- `crt_payload` (String) Decoded PEM certificate body (read-only).
- `csr_payload` (String) Decoded PEM CSR (read-only).
- `id` (String) UUID of the certificate.
- `in_use` (String) Whether this certificate is currently referenced by another OPNsense subsystem (`0`/`1`, read-only).
- `is_user` (String) Whether this certificate is linked to a user account (`0`/`1`, read-only).
- `name` (String) Distinguished name string of the certificate (read-only).
- `prv_payload` (String, Sensitive) Decoded PEM private key (read-only, sensitive).
- `ref_id` (String) Short hex reference ID used by other OPNsense subsystems to reference this certificate.
- `valid_from` (String) Unix timestamp of certificate start date (read-only).
- `valid_to` (String) Unix timestamp of certificate expiry date (read-only).
