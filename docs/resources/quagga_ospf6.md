# opnsense_quagga_ospf6 (Resource)

~> **Singleton Resource** This resource manages existing upstream Quagga configuration that cannot be created or destroyed through Terraform. You **must** import it before managing. Running `terraform destroy` removes the resource from state only and does **not** modify the upstream configuration.

Manages Quagga OSPFv3 (Open Shortest Path First version 3 for IPv6) global settings. This is a singleton resource that manages existing upstream configuration.

**Important:** This resource must be imported before it can be managed:
```bash
terraform import opnsense_quagga_ospf6.ospf6 quagga_ospf6
```

After importing, you can manage the configuration with `terraform apply`. Running `terraform destroy` will remove the resource from state but will NOT modify the upstream configuration.

## Example Usage

```terraform

resource "opnsense_quagga_ospf6" "ospf6" {
  enabled   = true

}
```

## Schema

### Optional

- `carp_demote` (Boolean) Increase the CARP demotion counter while OSPFv3 is not fully converged, causing CARP to prefer the peer until OSPFv3 is ready. Defaults to `false`.
- `enabled` (Boolean) Enable the OSPFv3 routing daemon. Defaults to `false`.
- `originate` (Boolean) Originate and advertise an OSPFv3 default route into the OSPFv3 domain. Defaults to `false`.
- `originate_always` (Boolean) Always originate the default route even when no default route exists in the routing table. Requires `originate` to be `true`. Defaults to `false`.
- `originate_metric` (Number) Metric value assigned to the originated default route. Use `-1` to leave unset.
- `router_id` (String) OSPFv3 router ID in IPv4 dotted-decimal notation (e.g. `"10.0.0.1"`). Leave empty to use the highest interface IP address.

### Read-Only

- `id` (String) Always set to `quagga_ospf6`. Use this value when importing: `terraform import opnsense_quagga_ospf6.ospf6 quagga_ospf6`
