# opnsense_quagga_rip (Resource)

~> **Singleton Resource** This resource manages existing upstream Quagga configuration that cannot be created or destroyed through Terraform. You **must** import it before managing. Running `terraform destroy` removes the resource from state only and does **not** modify the upstream configuration.

Manages Quagga RIP (Routing Information Protocol) settings. This is a singleton resource that manages existing upstream configuration.

**Important:** This resource must be imported before it can be managed:
```bash
terraform import opnsense_quagga_rip.rip quagga_rip
```

After importing, you can manage the configuration with `terraform apply`. Running `terraform destroy` will remove the resource from state but will NOT modify the upstream configuration.

## Schema

### Optional

- `default_metric` (Number) Default metric assigned to redistributed routes. Use `-1` to leave unset.
- `enabled` (Boolean) Enable the RIP routing daemon. Defaults to `false`.
- `networks` (Set of String) Set of network prefixes (CIDR notation) to enable RIP on. RIP will send and receive updates on interfaces that belong to these networks.
- `passive_interfaces` (Set of String) Set of interface names to make passive. Passive interfaces do not send RIP updates but still advertise their connected networks.
- `redistribute` (Set of String) Set of routing protocols whose routes should be redistributed into RIP. Valid values: `"bgp"`, `"connected"`, `"kernel"`, `"ospf"`, `"static"`.
- `version` (String) RIP protocol version to use. `"1"` uses broadcast updates; `"2"` uses multicast updates and supports CIDR. Defaults to `"2"`.

### Read-Only

- `id` (String) Always set to `quagga_rip`. Use this value when importing: `terraform import opnsense_quagga_rip.rip quagga_rip`
