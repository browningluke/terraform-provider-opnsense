# opnsense_quagga_bgp (Resource)

~> **Singleton Resource** This resource manages existing upstream Quagga configuration that cannot be created or destroyed through Terraform. You **must** import it before managing. Running `terraform destroy` removes the resource from state only and does **not** modify the upstream configuration.

Manages Quagga BGP (Border Gateway Protocol) global settings. This is a singleton resource that manages existing upstream configuration.

**Important:** This resource must be imported before it can be managed:
```bash
terraform import opnsense_quagga_bgp.bgp quagga_bgp
```

After importing, you can manage the configuration with `terraform apply`. Running `terraform destroy` will remove the resource from state but will NOT modify the upstream configuration.

## Schema

### Optional

- `as_number` (String) The local Autonomous System (AS) number for this BGP instance (e.g. `"65000"`). Defaults to `"65551"`.
- `best_path` (Set of String) BGP best-path selection options to enable. Valid values: `"as-path confed"`, `"as-path multipath-relax"`, `"compare-routerid"`, `"peer-type multipath-relax"`, `"aigp"`, `"med missing-as-worst"`.
- `distance` (Number) Administrative distance applied to BGP routes. Use `-1` to leave unset. Valid range is 1–255.
- `enabled` (Boolean) When enabled, the BGP routing daemon is active. Defaults to `false`.
- `enforce_first_as` (Boolean) When enabled, requires that the first AS in the AS_PATH of received BGP updates matches the peer AS number. Defaults to `true`.
- `graceful` (Boolean) When enabled, BGP peers are notified of a planned restart and routes are preserved during the restart period. Defaults to `false`.
- `log_neighbor_changes` (Boolean) When enabled, logs a message whenever a BGP neighbor transitions between established and idle states. Defaults to `false`.
- `maximum_paths` (Number) Maximum number of equal-cost EBGP paths to install in the routing table for multipath load balancing. Use `-1` to leave unset.
- `maximum_paths_ibgp` (Number) Maximum number of equal-cost IBGP paths to install in the routing table for multipath load balancing. Use `-1` to leave unset.
- `network_import_check` (Boolean) When enabled, verifies that the networks configured via `networks` are present in the routing table before advertising them. Defaults to `true`.
- `networks` (Set of String) List of network prefixes (CIDR notation) to originate and advertise via BGP.
- `router_id` (String) BGP router ID in IPv4 dotted-decimal notation (e.g. `"10.0.0.1"`). Leave empty to use the highest interface IP address.

### Read-Only

- `id` (String) Always set to `quagga_bgp`. Use this value when importing: `terraform import opnsense_quagga_bgp.bgp quagga_bgp`
